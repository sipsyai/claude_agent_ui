# Creating a Request from a Service Item

## Overview

A Request created using a Service Item includes several key features:

- Pre-filled information
- Dedicated Workflow and SLA
- Own Approval Workflow
- Defined Task(s), sometimes organized into stages

## Prerequisites

Before creating requests from Service Items, ensure the following have been established:

- A [Service Category](/serviceops-docs/technician-user-guide/admin-section/service-catalog/create-a-service-category)
- A [Service Template](/serviceops-docs/technician-user-guide/admin-section/service-catalog/create-a-service-catalog-template)

## How to Create a Request from a Service Item

### Example Scenario

An HR department employee initiates onboarding for a new Marketing department staff member and needs to create credentials through the Customer Portal using a Service Item.

### Steps to Follow

1. **Access the Portal**: Log into the Customer Portal and click the **Create New +** button on the Dashboard
2. **Select Service Request**: Choose the **Request Service** option
3. **Choose Category and Item**: Click the HR category and select the "Employee On-Boarding (IT)" item
4. **Complete the Form**: Fill in all required fields in the form that appears
5. **Submit**: Create the Request

### Request Status

Once created, the Request status will be one of the following:

- Open
- In Progress
- Pending
- Resolved
- Closed

## Additional Features

You can view audit logs for any created request to track changes and activities throughout its lifecycle.
